Convert LAS to MultiPointZ shapefile (LAS2MultiPointZShape)

This tool can be used to convert a LAS file, containing LiDAR data, into an equivalent vector shapefile of a MultiPointZ shape type. Because MultiPointZ shapefiles store all points within a single record, this is a far more efficient way of storing and visualizing the many millions of LiDAR points often contained within LAS files. Information about point Z and Intensity data are stored in the Z and Measure components of the Shapefile data. This tool provides a more efficient alternative to the Convert LAS to Shapefile (LAS2Shapefile) tool, which stores each LiDAR point in a separate point record, along with a cooresponding entry in the database file (.dbf). If you need the output file to contain data for the point classification values (CLASS), return numbers (RTN_NUM), and number of returnss (NUM_RTNS), the Convert LAS to Shapefile tool may be a better choice.

Scripting:

The following is an example of a Python script using this tool:

wd = pluginHost.getWorkingDirectory()
inputFile = wd + "input1.las"
outputFile = wd + "output.shp"
args = [inputFile, outputFile]
pluginHost.runPlugin("LAS2MultiPointZShape", args, False)

This is a Groovy script also using this tool:

def wd = pluginHost.getWorkingDirectory()
def inputFile = wd + "input.las"
def outputFile = wd + "output.shp"
String[] args = [inputFile, outputFile]
pluginHost.runPlugin("LAS2MultiPointZShape", args, false)
